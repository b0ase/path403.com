## What We Test

**WCAG 2.1 Level AA Compliance:**

### Perceivable
- **Text Alternatives**: Images have alt text
- **Captions**: Videos have captions/transcripts
- **Adaptable**: Content works with assistive tech
- **Distinguishable**: Sufficient color contrast (4.5:1 minimum)

### Operable
- **Keyboard**: All features keyboard accessible
- **Enough Time**: No time limits or adjustable
- **Seizures**: No flashing content >3x/second
- **Navigable**: Clear navigation, skip links
- **Input**: Forms have clear labels

### Understandable
- **Readable**: Clear language, proper headings
- **Predictable**: Consistent navigation
- **Input Assistance**: Form error messages

### Robust
- **Compatible**: Works with assistive tech
- **Valid HTML**: Proper markup
- **ARIA**: Correct usage of ARIA attributes

## Common Issues We Find

**Visual Issues:**
- Low color contrast (text hard to read)
- Missing image alt text
- Unlabeled form inputs
- Unclear link text ("click here")
- Missing heading hierarchy
- Content relies on color alone

**Interaction Issues:**
- Keyboard traps (can't tab out)
- No focus indicators
- Inaccessible modals/dropdowns
- Buttons that aren't buttons (<div> instead of <button>)
- No skip navigation link
- Inaccessible carousels

**Content Issues:**
- Jargon without explanation
- Complex language
- Time-limited content
- Auto-playing videos/audio
- PDF files not accessible

## Severity Levels

**Critical** (Must fix immediately)
- Blocks disabled users completely
- Legal liability risk
- Example: Forms unusable with keyboard

**High** (Fix urgently)
- Significantly impacts usability
- WCAG Level A failures
- Example: Missing alt text on important images

**Medium** (Fix soon)
- Impacts some users
- WCAG Level AA failures
- Example: Color contrast 3.8:1 (should be 4.5:1)

**Low** (Improve quality)
- Minor usability impact
- Best practice recommendations
- Example: Heading skip from H1 to H3
